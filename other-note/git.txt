1. Git基本配置
git config可以配置git的参数
使用git config --list查看已经配置的git参数。
有三个级别的保存位置，--system、--global、--local，分别表示所有用户（本系统）、当前用户（全局）、本地配置（当前目录），默认使用--local。

配置用户名及邮箱
在使用Git提交前，必须配置用户名和邮箱，这些信息会永久保存到历史记录中。
git config --global user.name "Tocy"
git config --global user.email zyvj@qq.com

初始化当前目录，即创建Git仓库 : git init

Git中每个文件都有三种状态：committed、staged、modified。它们之间关系如下：
commit <-- stage <-- modify
commit --> --- --modify

git status

获取远端库最新信息
git fetch origin
做diff
git diff xxx origin/xxx

git log 
 --graph     # 图像化分支和版本更新

用git remote查看当前的远程库。

更新
git pull =
git fetch + git merge

查看远程仓库信息
git remote show origin

git clone -b 分支

git add -A  提交所有变化
git add -u  提交被修改(modified)和被删除(deleted)文件，不包括新文件(new)
git add .   提交新文件(new)和被修改(modified)文件，不包括被删除(deleted)文件

bash下的提交过程
1 git add  -A 
2 git commit
3 git push 

git reset --soft --mix --hard

比较本地仓库和远程仓库的差异
git diff --stat 仓库名 origin

修改当前库的文件
git checkout --patch B库 文件名
