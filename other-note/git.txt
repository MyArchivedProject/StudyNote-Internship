1. Git基本配置
git config可以配置git的参数
使用git config --list查看已经配置的git参数。
有三个级别的保存位置，--system、--global、--local，分别表示所有用户（本系统）、当前用户（全局）、本地配置（当前目录），默认使用--local。

在使用Git提交前，必须配置用户名和邮箱，这些信息会永久保存到历史记录中。
git config --global user.name "Tocy"
git config --global user.email zyvj@qq.com

2. Git中每个文件都有三种状态：committed、staged、modified。它们之间关系如下：
commit <-- stage <-- modify
commit --> --- --modify
git status 查看基本工作目录的基本信息

3. 初始化当前目录，即创建Git仓库 : git init

4.  git remote show origin 显示所有远程的分支和相关信息
git fetch origin 获取远端库最新信息
git pull = git fetch + git merge  更新本地工作目录

5. git diff xxx origin/xxx  工作目录和缓冲区比较
--cache 缓存区和本地仓库比较
--HEAD 工作目录和本地仓库比较

git diff --stat 仓库名 origin   比较工作区和最新本地仓库的差异
git diff FETCH_HEAD 接比较fetch下来的内容和本地工作区的区别

6. git log 
 --graph     # 图像化分支和版本更新
 -p，用来显示每次提交的内容差异。 你也可以加上 -2 来仅显示最近两次提交：
git log -p -2
--pretty=oneline，short，，
git log -p + 文件名 （可查看该文件以前每一次push的修改内容）
git log - p -1 + 文件名 （只查看该文件当前这一次的push内容）

7. git clone -b 分支 拉取分支

8. git add -A  提交所有变化
git add -u  提交被修改(modified)和被删除(deleted)文件，不包括新文件(new)
git add .   提交新文件(new)和被修改(modified)文件，不包括被删除(deleted)文件

9. bash下的提交过程
1 git add  -A 
2 git commit
3 git push 

10. git reset --soft --mix --hard

11. git checkout --patch B库 文件名   修改当前库的文件
git checkout .   重置工作区

12. 将另一个分支的commit合并到此分支上。
git cherry-pick commit的id号

13. 删除远程的分支
git push origin :分支名 （没有删除跟踪）
git push origin --delete 分支名 （删除了跟踪）
git fetch -p 删除不存在的远程跟踪分支

14. git blame fileName 查看此文件内容详细信息
git push origin 本地分支名:远程分支名 创建新的远程分支并push上去
